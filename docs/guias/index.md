# Guías por rol

Un camino corto según quién eres. Cada guía te dice de qué eres **dueño**, qué **no tocas**,
y tu **día a día**.

- [**Guía del PO / funcional**](./po) — dueño del **QUÉ**: defines las User Stories y el
  porqué, nunca el CÓMO. Entrás por `grill-user-story`, `grill-epic` o `doc-to-backlog`.
- [**Guía del dev / ingeniero**](./dev) — dueño del **CÓMO** y del **link**: el agente
  implementa con `/opsx:apply`, tú revisas y autoras el `implements.yaml`.
- [**Guía del lead / SM / arquitecto**](./lead) — custodio de las **invariantes** y del
  **nivel de ceremonia** (N1 / N2 / N3): haces que el método se cumpla y se aligere donde
  corresponde.
